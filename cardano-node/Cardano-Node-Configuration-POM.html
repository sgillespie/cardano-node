  • ncExperimentalProtocolsEnabled :: !Bool

    During the development and integration of new network protocols (node-to-node and node-to-client) we wish to be able to test them but not have everybody use them by default on the mainnet. Avoiding enabling them by default makes it practical to include such not-yet-ready protocol versions into released versions of the node without the danger that node operators on the mainnet will start using them prematurely, before the testing is complete.

    The flag defaults to False

    This flag should be set to True when testing the new protocol versions.
